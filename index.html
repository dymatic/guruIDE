Welcome to the Guru Integrated Development Environment!

Some have argued that this program is not a true IDE. In its current state, I would agree that it barely qualifies for the title. However, the program is capable of generated code that the user did not create, so in that sense it is an IDE.

Contributing

I welcome contributions to this IDE. If you want to help you can clean up my source a bit, implement these changes, or write tutorials / documentation for me. Crash testers are also welcome.

I am also in dire need of artwork for my program. My current artwork is located in my project here. I made all of these icons in GIMP and they just look a little bit too... Cartoonish.

Support

I offer 1-on-1 support for guru here: email. You can also stop by my blog or IRC and leave a message regarding your problem. If the program is crashing, tell me what you were doing when it happened. If the problem is a button not doing anything (common) let me know which one it is and I will wire the connection. Graphical problems are on your end and may be caused by your Qt theme. I wrote all of my interfaces with the Lucidity GTK theme in mind.

Getting Guru IDE

Guru IDE is free software. Return to the code section and you will notice a .deb file. The .deb is a file installable on any debian-based system. The word "ubuntu" in the file name is just to keep track of what it was designed for. This will work on both Debian and Ubuntu as long as the dependencies are met. There is also a .rpm package for Fedora, Red-Hat, Opensuse, Mageia, mandriva, and any related system.

If you are NOT on a Debian system but still on Linux you can go into gDeb/usr/bin and download the executable. You will not be given any Window icons with this option.
